# KindleUnpack JavaScript Port

This is a complete JavaScript port of KindleUnpack based on Node.js, used to unpack Kindle ebooks in MOBI, PRC, AZW, AZW3, and AZW4 formats.

## Features

- Full support for unpacking MOBI, PRC, AZW, AZW3, and AZW4 ebooks
- Extract HTML/XHTML content from books
- Extract images (JPEG, PNG, GIF, BMP, SVG)
- Extract font files (TTF, OTF)
- Decompress PalmDoc compression format
- Decompress Huff/CDIC compression format
- Full recognition and processing of combination formats (Mobi 7 + KF8)
- Generate NCX table of contents and EPUB3 navigation document (nav.xhtml)
- Generate content.opf and EPUB files
- Support HD image extraction
- Support special handling for dictionary formats
- Support Fixed Layout / EPUB3 metadata

## Requirements

- Node.js 14.0 or higher
- Dependency: `adm-zip` (for EPUB generation)

Install dependencies:
```bash
npm install
```

## Usage

### Basic Usage

```bash
node kindleunpack.js <input_file> [output_directory]
```

### Examples

```bash
# Basic unpack
node kindleunpack.js book.mobi

# Specify output directory
node kindleunpack.js book.mobi ./output
```

### DumpMobiHeader Tool

```bash
node DumpMobiHeader_v023.js book.mobi
```

## Output Structure

Unpacked files are organized as follows:

```
<output_directory>/
├── mobi7/              # Mobi 7 format content
│   ├── book.html       # Main HTML file
│   ├── Images/         # Extracted images
│   └── content.opf     # Metadata file
├── mobi8/              # KF8 format content (if present)
│   ├── META-INF/
│   │   └── container.xml
│   └── OEBPS/
│       ├── Images/
│       ├── Fonts/
│       ├── Styles/
│       ├── Text/       # XHTML files
│       ├── content.opf
│       └── toc.ncx
│   └── <book_title>.epub    # Generated EPUB file
└── HDImages/           # HD images (if present)
```

## Project Structure

All Python modules under the `lib/` directory have been fully translated to `lib_js/`:

- `compatibility_utils.js` - Cross-platform compatibility utilities
- `imghdr.js` - Image format detection
- `kindleunpack.js` - Core unpack orchestrator
- `mobi_cover.js` - Cover processing
- `mobi_dict.js` - Dictionary support
- `mobi_header.js` - MOBI header parser
- `mobi_html.js` - HTML post-processing
- `mobi_index.js` - INDX index parser
- `mobi_k8proc.js` - KF8 processor
- `mobi_k8resc.js` - RESC processor
- `mobi_nav.js` - EPUB3 navigation document generator
- `mobi_ncx.js` - NCX table of contents generator
- `mobi_opf.js` - OPF generator
- `mobi_pagemap.js` - Page map processor
- `mobi_sectioner.js` - Palm DB section parser
- `mobi_split.js` - Combo MOBI splitter
- `mobi_uncompress.js` - Decompression algorithms (PalmDoc, Huff/CDIC)
- `mobi_utils.js` - General utilities
- `mobiml2xhtml.js` - MobiML to XHTML converter
- `unipath.js` - Path utilities
- `unpack_structure.js` - Output directory structure manager

## Relationship with Python Version

This project translates each module in the Python `lib/` directory to JavaScript, placed in the `lib_js/` directory. The JS version produces output identical to the Python version (verified by comparing actual MOBI file outputs).

Main differences:
1. Uses Node.js `Buffer` class for binary data handling
2. Uses `fs` module for file operations
3. Uses `adm-zip` instead of Python `zipfile` to generate EPUB

## Notes

- Only supports unencrypted ebooks
- For DRM-protected books, DRM must be removed first
- Due to metadata differences between `adm-zip` and Python `zipfile`, the `.epub` file generated by the JS version may be about 10–15KB larger than the Python version, but the extracted internal files are completely identical
